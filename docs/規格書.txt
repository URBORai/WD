========================================
電子喜帖網站 - 專案規格書 v3.0
========================================

### I. 專案總結：電子喜帖網站開發計畫

本專案是一個現代化、具高度互動性的單頁式（Single-Page）電子喜帖網站。

核心特色：
- 精緻的視覺動畫與滾動效果
- 強制性的背景音樂體驗（淡入播放）
- 完整的婚禮資訊展示（日期、地點、進程）
- 實用的功能性（RSVP 表單連結、Google Maps 地圖指引）
- 彈性化的配置系統（config.js），方便快速修改網站內容
- 可部署於 Microsoft Azure 靜態網站服務

### II. 功能規格 (Feature Specifications)

本專案已實現以下完整功能：

1.  **專案結構**
    - 單頁式網站設計 (Single-Page Application)
    - 響應式佈局，支援各種螢幕尺寸
    - 模組化配置系統 (config.js)

2.  **開發與部署環境**
    - 開發工具：Visual Studio Code (VS Code)
    - 即時預覽：Live Server 擴充套件
    - 部署平台：Azure Static Web Apps（或任何靜態網站託管服務）

3.  **導覽系統**
    - 固定式/黏貼式 (Sticky) 導覽列，恆定於頁面頂部
    - 導覽列包含 5 個錨點連結：首頁、關於我們、婚禮進程、回覆出席、地點
    - 平滑滾動 (Smooth Scroll) 跳轉效果
    - 自動修正錨點偏移，避免被導覽列遮擋
    - ✨ 滾動自動高亮功能：滾動頁面時自動高亮對應的導航項目

4.  **網站內容區塊**
    - **首頁 (Hero Section):** 新人名字、婚禮日期、歡迎標語
    - **關於我們 (About):** 新人故事簡介、婚禮日期與時間資訊
    - **婚禮進程 (Timeline):** 垂直時間軸，展示婚禮當日流程
    - **回覆出席 (RSVP/Links):** 兩張互動卡片
      · 卡片一：連結至 Microsoft 表單 (RSVP)
      · 卡片二：連結至舊版喜帖
    - **地點 (Venue):** ✨ 新設計
      · 兩個定位按鈕（真耶穌教會板橋教會、天賜良緣）
      · 點擊按鈕可切換下方地圖預覽
      · Google Maps 嵌入地圖（可互動、縮放、拖曳）
    - **頁尾 (Footer):** 結尾感謝訊息

5.  **滾動動畫 (AOS - Animate On Scroll)**
    - 使用 AOS 函式庫實現滾動觸發動畫
    - 支援淡入 (fade-up, fade-left, fade-right)、縮放 (zoom-in) 等效果
    - 可自訂動畫持續時間、延遲、觸發偏移量

6.  **背景音樂系統**
    - **觸發機制：** 頁面載入後自動彈出 SweetAlert2 對話框
    - **使用者互動：** 對話框僅提供一個「確定」按鈕（如：「開始體驗」）
    - **安全限制：** 不允許點擊外部關閉，確保使用者授權播放
    - **播放效果：** 點擊確定後，音樂以「淡入」(Fade In) 效果播放
    - **循環播放：** 音樂持續循環 (loop)，直到網頁關閉
    - **音量控制：** 可透過 config.js 設定最大音量與淡入時間

7.  **Google Maps 整合**
    - 嵌入式互動地圖
    - 標示婚宴場地位置
    - 支援縮放、拖曳等原生地圖操作

8.  **配置系統 (config.js)** ⭐ 核心特色
    - 集中管理所有網站內容，無需修改 HTML/CSS
    - 可配置項目：
      · 新人名字
      · 婚禮日期、時間、星期
      · ✨ 地點地圖網址（兩個地點：教會、餐廳）
      · 外部連結 (RSVP 表單、舊版喜帖)
      · 背景音樂檔案、音量、淡入時間
      · 所有文案內容
      · SweetAlert2 彈窗設定
      · AOS 動畫參數
      · 配色方案參考

### III. 設計風格 (Design Style)

本專案採用**「現代優雅極簡主義」**設計風格。

**視覺設計**
- 佈局：區塊式設計 (Block-based)，每個區塊承載獨立資訊
- 排版：使用 CSS Flexbox 和 Grid 進行精準對齊
- 留白：充足的空白空間，創造舒適的閱讀體驗
- 配色：金色主題 (#d4a574) 搭配白色與淺色背景，營造典雅質感

**互動設計**
- 顯性互動：
  · SweetAlert2 歡迎彈窗
  · 黏貼式導覽列的錨點跳轉
  · 可點擊的連結卡片
- 隱性互動：
  · AOS 滾動動畫（淡入、滑入、縮放效果）
  · 平滑滾動過渡
  · 音樂淡入效果

**響應式設計**
- 支援桌面、平板、手機等各種裝置
- 自動調整版面配置與字體大小

### IV. 技術架構 (Technology Stack)

**核心技術**
1.  HTML5 - 網頁結構與語義化標籤
2.  CSS3 - 視覺樣式與版面配置
3.  JavaScript (ES6+) - 互動邏輯與動態內容

**CSS 技術應用**
- Flexbox - 實現彈性佈局（如兩欄式連結卡片）
- CSS Grid - 複雜版面排列
- Position: Sticky - 固定式導覽列
- Scroll-behavior: smooth - 平滑滾動效果
- Scroll-margin-top - 修正錨點偏移問題
- CSS Variables - 主題色彩管理
- Media Queries - 響應式設計

**JavaScript 函式庫**
- AOS (Animate On Scroll) v2.3.1
  · 滾動觸發的動畫效果
  · CDN 引入方式
- SweetAlert2 v11
  · 美化的彈出式對話框
  · 用於音樂啟動授權

**外部服務整合**
- Google Maps Embed API
  · 嵌入式互動地圖
  · 無需 API Key（使用公開嵌入網址）

**開發工具**
- Visual Studio Code - 程式碼編輯器
- Live Server 擴充套件 - 本地開發伺服器與即時重載

**版本控制與部署**
- Git - 版本控制
- GitHub / Azure DevOps - 程式碼儲存庫
- Azure Static Web Apps - 靜態網站託管服務（支援 CI/CD）

**檔案結構**
```
wedding-invitation/

```

### V. 使用指南 (User Guide)

**快速開始（針對非技術使用者）**

1. **修改網站內容**
   - 開啟 `config.js` 檔案
   - 找到要修改的項目（如新人名字、婚禮日期等）
   - 修改引號內的文字
   - 儲存檔案，重新整理瀏覽器即可看到變更

2. **必須修改的項目**
   - 新人資訊：CONFIG.couple.groom, CONFIG.couple.bride
   - 婚禮日期：CONFIG.wedding.date, CONFIG.wedding.dayOfWeek, CONFIG.wedding.time
   - 地點地圖：CONFIG.maps.church, CONFIG.maps.restaurant
   - 外部連結：CONFIG.links.rsvpForm, CONFIG.links.oldInvitation

3. **本地測試**
   - 方法一：使用 VS Code + Live Server 擴充套件
   - 方法二：直接雙擊 index.html 開啟（部分功能可能受限）

**進階使用（針對開發者）**

1. **環境設定**
   ```bash
   # 安裝 VS Code
   # 安裝 Live Server 擴充套件
   # Clone 或下載專案
   ```

2. **自訂樣式**
   - 編輯 `style.css` 檔案
   - 修改顏色、字體、間距等視覺元素
   - 建議配合 `config.js` 中的 `colors` 參考

3. **替換圖片**
   - 準備您的照片檔案
   - 將圖片放置於專案資料夾
   - 在 `index.html` 中修改對應的 `<img src="...">` 路徑

4. **修改動畫效果**
   - 調整 `config.js` 中的 `aos` 設定
   - 或直接在 HTML 中修改 `data-aos` 屬性

5. **更換背景音樂**
   - 準備 .mp3 音樂檔案
   - 將檔案放入專案資料夾
   - 在 `config.js` 中修改 `CONFIG.music.filename`

### VI. 部署流程 (Deployment Guide)

**部署至 Azure Static Web Apps**

1. **前置作業**
   - 建立 GitHub 帳號並建立新的 Repository
   - 將專案檔案推送至 GitHub

2. **Azure 部署步驟**
   ```
   1. 登入 Azure Portal (https://portal.azure.com)
   2. 建立資源 → 選擇 "Static Web Apps"
   3. 填寫基本資訊：
      - 訂閱：選擇您的訂閱
      - 資源群組：建立新群組或選擇既有
      - 名稱：輸入網站名稱
      - 區域：選擇最近的區域
   4. 部署詳細資料：
      - 來源：選擇 GitHub
      - 組織：選擇您的 GitHub 帳號
      - 儲存庫：選擇專案儲存庫
      - 分支：選擇 main 或 master
   5. 建置詳細資料：
      - 建置預設：Custom
      - 應用程式位置：/（根目錄）
      - API 位置：留空
      - 輸出位置：留空
   6. 檢閱 + 建立 → 建立
   ```

3. **部署後設定**
   - Azure 會自動執行 GitHub Actions 進行部署
   - 部署完成後，取得 Azure 提供的網址
   - 測試網站所有功能是否正常運作

4. **Google Maps 設定（如需 API Key）**
   - 若使用嵌入式地圖，通常不需要 API Key
   - 若需要，請至 Google Cloud Console 設定 HTTP 參照網址限制

**其他部署選項**
- GitHub Pages（免費）
- Netlify（免費方案）
- Vercel（免費方案）
- 任何支援靜態網站的託管服務

### VIII. 常見問題 (FAQ)

**Q1: 為什麼音樂沒有播放？**
A:
- 確認音樂檔案名稱與 config.js 設定相符
- 確認音樂檔案與 index.html 在同一資料夾
- 瀏覽器可能阻擋自動播放，請點擊彈窗的「開始體驗」按鈕
- 檢查瀏覽器是否靜音

**Q2: 如何修改顏色？**
A:
- 編輯 `style.css` 檔案
- 搜尋顏色代碼（如 #d4a574）並替換為您想要的顏色
- 建議同步更新 `config.js` 中的 `colors` 物件作為參考

**Q3: 地圖沒有顯示？**
A:
- 確認 Google Maps 嵌入網址是否正確
- 檢查網址是否有權限限制
- 測試是否為網路連線問題

**Q4: 修改 config.js 後沒有變化？**
A:
- 按 Ctrl+Shift+R (Windows) 或 Cmd+Shift+R (Mac) 強制重新整理
- 清除瀏覽器快取

**Q5: 如何取得 Google Maps 嵌入網址？**
A:
方法一（推薦）：使用簡易格式
- https://www.google.com/maps?q=新北市板橋區大觀路2段2號&output=embed

方法二：使用 Google Maps 嵌入網址
1. 前往 Google Maps (https://maps.google.com)
2. 搜尋地點
3. 點擊「分享」→「嵌入地圖」
4. 點擊「複製 HTML」
5. 從代碼中找到 src="..." 部分的網址
6. 貼上至 CONFIG.maps.church 或 CONFIG.maps.restaurant

**Q6: 如何修改地點按鈕的名稱？**
A:
- 需要編輯 index.html 檔案
- 找到地點區塊（id="venue"）中的按鈕部分
- 修改按鈕文字即可

**Q7: 地點按鈕點擊後地圖沒有切換？**
A:
- 確認 config.js 中的 maps.church 和 maps.restaurant 網址正確
- 打開瀏覽器開發者工具 (F12) 查看是否有錯誤訊息
- 確認網址格式正確（應包含 &output=embed 或使用 Google Maps embed URL）

### IX. 技術支援與維護

**除錯方法**
- 開啟瀏覽器開發者工具（F12）
- 檢查 Console 是否有錯誤訊息
- 確認所有檔案路徑正確
- 檢查 config.js 語法（引號、逗號、括號是否配對）

**效能優化建議**
- 壓縮圖片檔案（使用 TinyPNG 等工具）
- 使用適當的圖片格式（JPG 適合照片，PNG 適合圖示）
- 考慮使用 lazy loading 延遲載入圖片

**安全性注意事項**
- 不要在程式碼中包含敏感資訊
- 使用 HTTPS 部署（Azure Static Web Apps 自動提供）
- 定期更新 CDN 函式庫版本
